# 03 映射系统

映射系统是底层基座的核心。它解决从“我要达成目标”到“今天做什么”的推导过程。

关键反驳：

```text
不要做成：目标 -> 动作
应该做成：目标 -> 指标 -> 数据 -> 卡点 -> 动作 -> 验证
```

如果目标直接映射动作，系统会很快变成死规则。比如“回复率低就开提醒”，但真实原因可能是排班空档、低质量询盘、快捷回复不足、晚间无人接待或客服被售后占用。

## 1. 目标-指标映射

回答：某个目标由哪些指标决定。

| 目标 | 关联指标 | 备注 |
| --- | --- | --- |
| 找工厂牌级 | 旺旺 3 分钟响应率、找工厂服务响应率、找工厂履约率、定制交易积分、合约支付率、月动销小单定制商品数 | 官方规则必须版本化。 |
| 服务体验修复 | 3 分钟响应率、退款处理时长、品质退款率、品质问题率、平台介入率、48 小时揽收率 | 先保基础体验。 |
| 店铺 L 等级成长 | GMV、买家数、响应率、揽收率、老客回头率、潜力商品数、采购指数4.0商品数、特色货盘 | 具体口径按官方更新；金冠品数降级为历史/过渡字段。 |
| 成交增长 | 曝光、访客、询盘、支付、支付金额、毛利率、广告消耗 | 毛利和履约是增长前置条件。 |
| 员工能力建设 | 动作完成率、有效动作数、停止无效动作数、SOP 沉淀数 | 不直接等于销售额。 |

## 2.1 动作频率映射

回答：动作是一次性配置、周期性巡检，还是异常触发。这个字段决定系统是否每天派任务。

| 动作类型 | 定义 | 示例 | 派发规则 |
| --- | --- | --- | --- |
| `one_time_setup` | 配置完成后不应重复派发 | 开通买家保障、开通发货保障、设置基础服务、补商品必填属性 | 未完成时派发；完成后只保留状态 |
| `periodic_check` | 需要按固定周期复核 | 每周看新灯塔、每周看镇店之宝配额、每月看规则版本 | 到周期才派发 |
| `daily_operation` | 日常经营动作 | 回复询盘、处理退款、待发货日清、报价追单 | 每日按异常和目标派发 |
| `exception_triggered` | 只有异常时才派发 | 保障违约率上升、星级下降、退款处理超时、物流异常 | 指标越线才派发 |
| `experiment` | 需要观察结果的优化动作 | 改标题、改主图、优化详情页、调整报价模板 | 设验证周期，到期回测 |

买家保障已经开通后，应从 `one_time_setup` 转为 `periodic_check` + `exception_triggered`：系统不再每天提醒“去开通保障”，只在保障缺失、承诺违约、服务分下滑或商品新增时提醒。

## 3. 指标-卡点映射

回答：指标异常可能代表什么经营卡点。

| 指标异常 | 可能卡点 | 不能立刻下结论 |
| --- | --- | --- |
| 曝光低 | 流量卡点、标题关键词不足、商品覆盖不足、类目不准 | 不一定要马上加广告。 |
| 广告曝光占比高 | 自然流量弱、投放依赖高 | 不一定是广告错，可能商品自然承接弱。 |
| 访客低 | 点击卡点、主图/价格/标题不吸引 | 先看曝光是否足够。 |
| 访客有但询盘低 | 商品信任卡点、详情页信息不足、采购门槛高 | 不一定是客服问题。 |
| 询盘有但支付低 | 报价、交期、样品、定制流程、客服跟进卡点 | 不一定要降价。 |
| 3 分钟响应率低 | 接待机制卡点 | 要继续判断是排班、提醒、话术还是询盘质量。 |
| 履约率低 | 交付能力卡点、定制承诺过度、供应链不稳 | 不适合继续放大定制订单。 |
| 品质退款率高 | 商品质量、描述承诺、批次问题 | 不适合继续推广问题商品。 |
| 毛利率低 | 价格、成本、广告、售后、样品策略问题 | 不应该盲目冲 GMV。 |
| 采购指数 < 4.0 | 商品口碑、价格、服务、品质或商誉未达扶持门槛 | 不应该只改标题，要先看商品成长 AI 诊断。 |
| 旧金冠品但采购指数 < 4.0 | 过渡期权益即将失效 | 需要优先修采购指数短板，避免缓冲期后降级。 |

## 4. 卡点-动作映射

回答：某类卡点可以派哪些动作。

| 卡点 | 优先动作 | 负责人 |
| --- | --- | --- |
| 接待机制卡点 | 客服值班、3 分钟提醒、快捷回复、未响应归因 | 客服员工 |
| 商品信任卡点 | 补材质、容量、保温时长、拿样、定制、开票、交期、包装 | 运营员工 |
| 点击卡点 | 主图、标题前 15 字、价格梯度、场景图 | 运营员工 |
| 报价成交卡点 | 三档报价、样品政策、Logo 定制报价、二次跟进 | 客服员工/负责人 |
| 交付卡点 | 待发货日清、库存校准、定制确认稿、供应链确认 | 负责人 |
| 售后品质卡点 | 退款原因分类、问题 SKU 停推、详情页预期修正 | 负责人/运营员工 |
| 利润卡点 | 单品毛利表、低毛利订单标记、广告准入规则 | 负责人 |
| 商品成长卡点 | 按采购指数五维度拆解口碑、价格、服务、品质、商誉，并修商品表达和履约短板 | 运营员工/负责人 |

## 5. 动作-验证映射

回答：动作做完以后看什么。

| 动作类型 | 验证指标 | 验证周期 | 成功标准 |
| --- | --- | --- | --- |
| 客服响应动作 | 3 分钟响应率、未响应消息数 | 明天/3 天 | 响应率上升，未响应原因减少。 |
| 商品信任动作 | 商品询盘数、访客询盘率 | 3-7 天 | 询盘率上升。 |
| 标题主图动作 | 访客数、点击承接 | 3-7 天 | 访客上升且询盘不下降。 |
| 报价动作 | 报价后成交率、合约支付率 | 7 天 | 追单后支付或明确流失原因。 |
| 交付动作 | 履约率、48 小时揽收率、退款率 | 7-30 天 | 风险订单下降。 |
| 售后动作 | 退款处理时长、品质退款率、介入率 | 7-30 天 | 处理时长或问题率下降。 |
| 毛利动作 | 毛利率、广告费率 | 7 天 | 毛利回到底线以上。 |
| 商品成长动作 | 采购指数、采购指数4.0商品数、镇店之宝可配商品数 | 7-30 天 | 商品采购指数达到或接近 4.0，或明确短板维度。 |
| 一次性配置动作 | 配置状态、配置覆盖率、异常触发数 | 完成后月度复核 | 已完成则不再派日常任务；出现新增商品、规则变化或违约异常时再触发。 |

## 当前结论

内置映射表未来可以做，但现在先用 Markdown 固化逻辑。后续软件化时，优先顺序应该是：

```text
指标字典
数据源映射
目标-指标映射
指标-卡点映射
卡点-动作映射
动作-验证映射
```
