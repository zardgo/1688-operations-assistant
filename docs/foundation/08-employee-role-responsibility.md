# 08 员工角色与责任模型

员工角色与责任模型回答一个问题：系统生成动作后，应该由谁做，谁确认，谁复盘。

如果没有角色模型，作战单会变成“大家都能做、最后没人负责”的任务池。长期看，系统必须区分客服、运营、负责人和外部协作者的责任边界。

## 角色定义

| 角色 | 核心职责 | 不应该承担 |
| --- | --- | --- |
| 客服员工 | 首响、询盘分层、快捷回复、报价跟进、未成交客户二次跟进。 | 商品结构、毛利底线、官方规则确认。 |
| 运营员工 | 商品标题、主图、详情页、商品信任点、数据导入、商品列表维护。 | 定制交期承诺、低毛利大单决策。 |
| 负责人 | 目标选择、规则确认、毛利底线、交付风险、异常升级、复盘裁决。 | 每天替员工完成常规录入和常规动作。 |
| 美工/内容协作者 | 主图、详情页、场景图、素材优化。 | 经营指标判断和目标选择。 |
| 仓储/供应链协作者 | 库存、发货、交期、质量问题反馈。 | 客服话术和商品流量优化。 |

## 指标责任映射

| 指标 | 主责 | 协作 | 负责人介入条件 |
| --- | --- | --- | --- |
| 3 分钟响应率 | 客服员工 | 负责人 | 连续低于目标 3 天。 |
| 找工厂服务响应率 | 客服员工 | 负责人 | 影响冲牌级或掉级。 |
| 商品曝光 | 运营员工 | 美工/内容 | 连续 7 天无改善。 |
| 商品访客 | 运营员工 | 美工/内容 | 主推商品点击持续差。 |
| 访客询盘率 | 运营员工 | 客服员工 | 商品有访客但连续无询盘。 |
| 询盘转支付 | 客服员工 | 负责人 | 涉及价格、交期、定制风险。 |
| 合约支付率 | 客服员工 | 负责人 | 高意向客户未支付或大单卡住。 |
| 履约率 | 负责人 | 仓储/供应链 | 定制交期或现货发货异常。 |
| 品质退款率 | 负责人 | 运营/供应链 | 涉及问题 SKU 停推。 |
| 毛利率 | 负责人 | 运营/客服 | 低于底线时停止增长动作。 |
| SOP 沉淀数 | 负责人 | 全员 | 周复盘未产出 SOP。 |

## 动作分配规则

动作分配应遵循：

```text
谁最能直接影响指标，谁主责。
谁拥有风险决策权，谁确认。
谁能沉淀为标准流程，谁复盘。
```

示例：

| 卡点 | 动作 | 主责 | 确认 |
| --- | --- | --- | --- |
| 响应率低 | 补值班和快捷回复 | 客服员工 | 负责人 |
| 有访客无询盘 | 补商品信任点 | 运营员工 | 负责人 |
| 询盘有但不成交 | 三档报价和二次跟进 | 客服员工 | 负责人 |
| 履约风险 | 定制订单交期排查 | 负责人 | 负责人 |
| 品质退款高 | 问题 SKU 停推 | 负责人 | 负责人 |
| 商品素材弱 | 主图/详情页优化 | 运营员工/美工 | 负责人 |

## 低摩擦执行原则

员工端不要要求复杂证明。

| 员工动作 | 推荐交互 |
| --- | --- |
| 今日动作完成 | 勾选完成。 |
| 有异常 | 写一句可选备注。 |
| 需要证据 | 只在高风险动作或复盘争议时补截图。 |
| 明日验证 | 系统根据数据判断，不让员工写长报告。 |

原因：

```text
员工的主要工作是改善指标，不是证明自己做了很多事。
```

## 负责人角色边界

负责人不是每天盯任务的人，而是经营裁判。

负责人需要做：

- 选择当前阶段目标。
- 确认官方规则版本。
- 设定毛利、履约、售后底线。
- 处理高风险异常。
- 每周判断动作是否进入 SOP。
- 停止无效动作。

负责人不应该长期做：

- 每天替员工录入所有数据。
- 每天替员工改商品。
- 每天替员工接待询盘。
- 用人工经验替代系统复盘。

如果负责人被日常执行拖住，系统要把它识别成能力或流程问题。

## 员工能力模型

员工能力不只看完成率。

| 能力 | 可观察信号 |
| --- | --- |
| 数据意识 | 能按时导入/录入关键数据，知道数据缺口。 |
| 执行能力 | 今日动作按时完成。 |
| 归因能力 | 能说清楚指标没动的可能原因。 |
| 停止能力 | 能停止无效动作，不为了忙碌继续做。 |
| SOP 能力 | 能把有效动作写成别人可复用的流程。 |

能力目标应该进入周复盘，而不是压到每日作战单里。

## 任务升级规则

| 情况 | 升级给谁 |
| --- | --- |
| 连续 3 天响应率无改善 | 负责人 |
| 低毛利订单影响增长判断 | 负责人 |
| 定制交期不确定 | 负责人/供应链 |
| 品质退款集中在某 SKU | 负责人/运营 |
| 员工连续完成任务但指标不动 | 负责人复盘 |
| 官方规则未确认但员工要冲目标 | 负责人确认规则 |

## 长期价值

角色模型让系统生成的不是“任务”，而是“有责任边界的作战单”：

```text
谁做
谁协作
谁确认
谁复盘
什么时候升级
```
