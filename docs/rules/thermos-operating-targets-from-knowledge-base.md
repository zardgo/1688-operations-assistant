## 4. 订单类型目标

| 订单类型 | 目标 | 关键指标 | 软件应生成的动作 |
| --- | --- | --- | --- |
| 现货批发 | 快速报价和稳定发货 | 48 小时揽收率、报价后成交率、毛利率 | 报三档阶梯价，确认库存和发货时效 |
| 拿样试单 | 判断真实后续采购可能 | 样品转大货率、响应率、回访率 | 标记样品客户，发货后 3/7 天回访 |
| LOGO 定制 | 让定制需求进入标准流程 | 合约支付率、履约率、定制交易积分 | 先问用途、数量、预算、交期，再走确认稿 |
| 礼品团购 | 控制大单交期和包装风险 | 客单价、毛利率、退款处理时长 | 给低中高三档方案，确认包装和交期红线 |
| 老客补货 | 提高复购和稳定现金流 | 老客回头率、复购率、采购周期 | 建客户档案，7 天回访，30 天补货提醒 |
| 渠道分销 | 建长期供货关系 | SKU 动销、毛利率、售后率 | 建分销货盘表，明确价格保护和售后边界 |
| 低质量询盘 | 减少客服消耗 | 有效询盘率、3 分钟响应率 | 快速筛选，不长聊，归档原因 |

## 6. 从样本到动作的规则

| 触发条件 | 诊断 | 今日 checklist |
| --- | --- | --- |
| 响应率低于目标 | 保温杯询盘承接弱，优先级高于改标题 | 补值班表、查未响应原因、补快捷回复 |
| 有曝光无访客 | 标题、主图、价格或人群词不匹配 | 优化 3 个主推款标题和主图 |
| 有访客无询盘 | 商品信任和采购入口不足 | 补开票、拿样、定制、交期、材质说明 |
| 询盘多但成交少 | 报价结构或客户分层不足 | 使用四问法和三档报价模板 |
| 样品多但大货少 | 拿样客户没有被跟进或质量不匹配 | 建样品客户表，3/7 天回访 |
| 老客少 | 每次都重新获客，经营不稳定 | 建老客表，30 天补货提醒 |
| 定制入口不足 | 找工厂和大单承接能力弱 | 补 3 个小单定制款和确认稿 SOP |
| GMV 增长但毛利不清 | 增长可能吞利润 | 补单品毛利表，低毛利单不硬追 |

## 7. 不应作为目标的内容

以下内容当前不能直接做成硬目标：

- 店铺商品数达到 500 款：本地文档已提示相关 CSV/修正数据不可靠。
- 金冠品数长期增长：金冠品体系已过期，只能用于 2026-05-25 至 2026-06-25 缓冲期和历史对比。
- 金牌制造立即达成：这是平台身份目标，当前只能作为长期对标，不应压员工短期动作。
- 单纯追 GMV：如果毛利、履约、售后未过线，GMV 增长会放大风险。
- 只看曝光：曝光没有转化为访客、询盘、成交和复购，就不是有效经营成果。

## 8. 软件落地建议

软件里应把目标分成三层：

1. 官方目标：找工厂牌级、服务体验、L 等级、新灯塔等，必须有规则版本和来源可信度，但不要求人工确认后才可使用。
2. 经营目标：响应、询盘、报价、交付、毛利、复购、SOP，由本文件支持。
3. 员工能力目标：快捷回复、报价模板、定制确认、老客回访、周复盘。

当三层目标冲突时，优先级为：

```text
服务/履约/售后底线 > 毛利底线 > 官方冲级 > GMV增长 > 商品扩张
```
