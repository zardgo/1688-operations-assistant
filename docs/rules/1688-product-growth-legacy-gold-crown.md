## 1. 使用边界

本文只用于理解旧商品成长体系和历史字段映射。

软件不得把金冠商品数作为长期核心目标，不得用旧成长分阈值驱动员工动作。当前长期商品成长目标应使用：

- 采购指数4.0商品数。
- 商品采购指数。
- 新品采购指数。
- 采购指数五维度短板。

## 2. 旧分层

| 旧分层 | 旧口径 | 当前处理 |
| --- | --- | --- |
| 不合规商品 | 商品成长总分为 0 | 转为采购指数体系下的不达标商品或后台当前状态。 |
| 普通商品 | 旧成长分大于 0 且小于 60 | 不作为当前目标。 |
| 潜力商品 | 旧成长分大于等于 60 且小于 100 | 需按采购指数体系重新确认。 |
| 金冠商品 | 旧成长分大于等于 100 | 已过期，只能用于历史对比和过渡期解释。 |

## 3. 旧评分维度

| 旧维度 | 旧资料说明 | 当前可迁移为 |
| --- | --- | --- |
| 品效表现 | 浏览表现、转化效率、回头率、成交规模等 | 商品口碑、成交、复购、搜索承接能力。 |
| 价格表现 | 价格真实性、批发价竞争力、代发价竞争力 | 采购指数价格维度。 |
| 内容表现 | 标题、主图、主图视频、商品讲解 | 商品信息质量、素材质量。 |
| 趋势表现 | 新发商品、商机趋势、破零速度 | 新品采购指数、新品成长动作。 |
| 服务表现 | 3 分钟旺旺响应率、48 小时揽收率、品质退款率、特色服务 | 服务、品质、履约维度。 |

## 4. 仍有参考价值的底层口径

虽然金冠体系过期，但旧资料里有几个底层经营口径仍值得保留为解释变量：

| 口径 | 保留方式 |
| --- | --- |
| 商品近 30 天表现更重要 | BI 看趋势时保留近 30 天窗口。 |
| 浏览、转化、复购、成交规模共同影响商品力 | 商品诊断不要只看曝光或成交。 |
| 商品转化效率和生意参谋口径可能不同 | 软件必须记录数据来源和口径。 |
| 价格作弊、起批量作弊、运费作弊会触发处罚 | 应拆成商品发布/价格合规红线。 |
| 标题、主图、视频、商品讲解影响内容质量 | 可进入商品优化 checklist。 |
| 服务表现引用新灯塔指标 | 服务体验规则需单独维护。 |

## 5. 历史字段映射

| 旧字段 | 新字段建议 | 说明 |
| --- | --- | --- |
| `gold_crown_product_count` | `purchase_index_4_product_count` | 长期目标替换。 |
| `product_growth_score` | `purchase_index_score` | 需以后台当前采购指数口径为准。 |
| `legacy_potential_product_count` | `potential_product_count` | 潜力商品仍需按新体系确认定义。 |
| `legacy_product_content_score` | `product_content_quality_status` | 可保留为商品信息质量诊断。 |
| `legacy_service_score` | `service_experience_status` | 转到新灯塔/服务体验规则。 |

## 6. 软件阻断规则

| 触发 | 系统行为 |
| --- | --- |
| 规则卡引用金冠商品数作为增长目标 | 阻断，并提示该字段已过期。 |
| 员工录入旧成长分 | 允许作为历史数据保存，但不能生成冲刺任务。 |
| 后台仍展示旧金冠字段 | 标记为历史/过渡字段，要求同时录入采购指数。 |
| 文档或计划出现“补金冠品数” | 替换为“修采购指数短板”。 |
