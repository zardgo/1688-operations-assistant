## 1. 可借鉴的核心判断

商品质量星级应从“查一个分数”改成“修一套发布质量和市场表现”。即使后台不展示清晰星级，软件也能帮助员工做可执行动作。

对保温杯店铺，商品质量可以拆成四个杠杆：

- 类目准确性；
- 标题专业性；
- 属性完整性；
- 服务保障完整性。

这四项适合做成商品编辑 checklist，而不是让员工追一个不可见分数。

## 2. 不应直接写死的说法

下列说法来自 AI 摘录，当前不能当作官方规则：

| 说法 | 处理方式 |
| --- | --- |
| 平台不提供星级查询入口 | 需要后台和规则中心验证 |
| 新灯塔体验分 >=90 默认质量星级已达 7 星 | 暂不作为规则 |
| 开通某服务售后分直接 +5 分 | 暂不作为规则 |
| 搜索曝光、点击率、加购率会按固定比例提升 | 不进入员工端承诺文案 |
| 3-5 天自动识别为高质高潜 | 只能作为复盘观察窗口假设 |

## 3. 软件字段建议

| 字段 | 含义 | 来源 | 动作方向 |
| --- | --- | --- | --- |
| `category_accuracy_status` | 类目是否准确 | 商品编辑页/诊断中心 | 修正错放类目 |
| `title_quality_status` | 标题是否清晰合规 | 商品编辑页/诊断中心 | 去堆砌词，补核心词和场景词 |
| `attribute_completion_rate` | 属性完整度 | 商品编辑页 | 补材质、容量、适用场景、定制能力等 |
| `service_protection_status` | 服务保障完整度 | 商品编辑页/服务保障 | 补履约和售后保障 |
| `product_search_exposure` | 搜索曝光 | 生意参谋商品分析 | 验证搜索匹配改善 |
| `product_click_rate` | 点击率 | 生意参谋商品分析 | 验证标题、主图、价格吸引力 |
| `product_add_to_cart_rate` | 加购率 | 生意参谋商品分析 | 验证详情页和利益点 |
| `product_pay_conversion_rate` | 支付转化率 | 生意参谋商品分析 | 验证商品力和交易承接 |

## 4. 员工 checklist

| 场景 | checklist |
| --- | --- |
| 商品无曝光 | 先查类目、标题关键词、属性是否完整 |
| 有曝光无点击 | 优化标题、主图、价格表达、核心卖点 |
| 有点击无转化 | 优化详情页、保障服务、起批量、定制说明 |
| 有成交但退款高 | 查品质、包装、承诺过度、售后原因 |
