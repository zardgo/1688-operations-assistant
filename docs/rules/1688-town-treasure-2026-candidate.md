## 1. 可借鉴的核心判断

镇店之宝不应只做“手动标记商品”的功能，而应被纳入核心商品资产模块。

对软件有价值的抽象是：

- 镇店之宝候选商品应来自商品能力、服务体验、交易表现和定制能力的综合判断。
- 它和 L 等级、新灯塔、采购指数4.0、哇噢定制、严选权益存在联动。
- 员工不应该只被要求“设置镇店之宝”，而应该知道“为什么这个商品有资格成为镇店之宝候选”。

## 2. 不应直接写死的说法

下列说法来自 AI 摘录，当前不能当作官方规则：

| 说法 | 处理方式 |
| --- | --- |
| 仅 L4 及以上商家可启用镇店之宝 | 需要后台或规则中心验证 |
| 系统每日凌晨自动更新商品池 | 需要后台页面验证 |
| 哇噢定制成交占比 >=20% 自动入围 | 暂做候选维度，不写死阈值 |
| 新灯塔物流体验 >=90 且售后体验 >=88 自动入围 | 暂做候选维度，不写死阈值 |
| 基础加权 +20%、L4 +5%、L5 +12%、定制每单 +0.8% | 不进入员工端承诺文案 |

## 3. 已有较高可信规则关系

用户已提供的 2026-05-09 商品成长升级公告中，镇店之宝配置门槛调整为采购指数 >=4.0 商品，且严选、供应链品牌、联营计划等可能影响配额来源。该部分已沉淀在：

- `1688-product-growth-purchase-index-2026-05-09.md`

因此软件第一版应优先采用“采购指数4.0 + 配额来源”作为镇店之宝基础判断，再把 AI 摘录中的 L 等级、新灯塔、定制能力作为候选排序维度。

## 5. 员工 checklist

| 场景 | checklist |
| --- | --- |
| 有配额未使用 | 先筛采购指数4.0商品，再看成交、转化、服务风险 |
| 候选商品不足 | 先修采购指数、服务保障、标题属性、价格竞争力 |
| 定制商品有潜力 | 补定制案例、确认稿、定制保障、交期说明 |
| 商品流量上升但售后风险高 | 暂缓放大，先修履约、品质、保障承诺 |
