# 1688 官方目标驱动的运营迭代驾驶舱产品设计

## 1. 产品定位

第一版产品不是普通报表、待办工具或 1688 ERP，而是给“一个人全包”的 1688 运营员工使用的迭代驾驶舱。

当前项目默认经营类目是 **保温杯**，应按消费品/日用百货类目处理。第一版所有默认指标、任务和示例都围绕保温杯店铺展开；其他类目只作为未来扩展。

它要把官方政策目标、店铺经营底线、每日动作和每周复盘串成一个闭环：

```text
选择官方目标
  -> 录入当前指标
  -> 系统诊断差距
  -> 生成今日任务
  -> 记录执行结果
  -> 每周复盘
  -> 沉淀 SOP
  -> 下轮继续冲级
```

产品的核心不是“让员工完成更多任务”，而是帮助员工在数据反馈中持续进步。

## 2. 目标宪法

系统必须先有决策优先级，否则会把员工引向指标作弊。

目标优先级如下：

1. **经营底线**
   不亏钱、不伤交付、不违规、不把老板拖进日常。

2. **平台保命**
   先保证旺旺 3 分钟响应、履约/揽收、退款处理、品质问题、介入率等基础服务指标，不掉权重，不影响活动报名和营销推广。

3. **官方成长**
   再冲找工厂牌级、L 等级、新灯塔体验分等平台成长目标。

4. **经营迭代**
   最后用选品、标题、报价、客服、交付、复购和 SOP 沉淀解释并修复前面的指标问题。

决策规则：

- 如果冲 GMV 会导致毛利过低，系统提示“不建议冲 GMV，先修毛利”。
- 如果响应率低，优先生成客服值班和 3 分钟响应任务，而不是让员工先改标题。
- 如果履约风险高，暂停推广和复杂定制单，先修交付。
- 如果官方指标达标但利润差，提示“平台表现好，但经营质量不合格”。
- 如果经营指标好但官方指标差，提示“打法有效，但平台权益风险高”。

## 3. 官方目标体系

### 3.1 找工厂牌级目标

来源：1688 规则中心「关于 2023 找工厂商家牌级体系升级公告」。

适用范围：已经入驻 1688 找工厂的商家，不含超级工厂。

| 目标牌级 | 近 30 天服务响应率 | 近 30 天履约率 | 近 30 天定制交易积分 | 近 30 天合约支付率 | 月动销小单定制商品数 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 铜牌工厂 | ≥60% | ≥70% | ≥10 万 | ≥50% | ≥3 款 |
| 银牌工厂 | ≥65% | ≥75% | ≥80 万 | ≥60% | ≥3 款 |
| 金牌工厂 | ≥70% | ≥80% | ≥150 万 | ≥70% | ≥3 款 |

对应权益包括找工厂搜索和推荐排名、必找工厂榜/安心购频道、工厂名片展示、镇店之宝配额、询报价单和营销活动权益。

### 3.2 新灯塔/服务管理目标

来源：1688 规则中心「1688 商家服务管理规范」。

平台从发货体验、咨询体验、售后体验等维度考核商家服务表现。服务表现会影响商品成长、商家等级、活动报名、搜索推荐等权益。

关键指标：

| 维度 | 指标 | 口径要点 |
| --- | --- | --- |
| 物流体验 | 近 30 天 48 小时揽收率 | 保温杯按消费品优先看该指标，剔除合约/定制/邀约等订单 |
| 物流体验 | 近 30 天履约率 | 找工厂牌级仍需关注；在本项目中作为冲找工厂牌级的专项指标 |
| 物流体验 | 近 30 天物流时效达成率 | 仅考核符合条件的现货且非无需物流订单 |
| 物流体验 | 近 30 天发货物流退款率 | 买家首次选择发货或物流相关原因退款/换货 |
| 咨询体验 | 近 30 天旺旺 3 分钟响应率 | 每日 09:00-21:00，考核买家咨询消息 3 分钟内有效回复 |
| 咨询体验 | 近 30 天旺旺咨询满意度 | 评价 4-5 分为满意 |
| 商品体验 | 近 30 天品质退款率 | 品质相关退款/换货子订单数 / 支付子订单数 |
| 商品体验 | 近 30 天品质问题率 | 评价或退款备注含品质问题的主订单数 / 支付主订单数 |
| 售后体验 | 近 30 天退款处理时长 | 退款申请到完结的卖家处理总时长 / 完结退款笔数 |
| 售后体验 | 近 30 天介入率 | 流入平台处理的投诉笔数 / 支付成交主订单数 |

旺旺 3 分钟响应率特别规则：

- 考核时间为每日 09:00-21:00。
- 有效回复包括人工回复、AI 客服有效答复等。
- 旺旺/千牛设置的普通自动回复属于无效回复。
- 店铺 3 分钟响应率低于 50% 时，平台可能自动启用平台机器人客服接待。

### 3.3 L 等级/店铺成长目标

来源：1688 规则中心「关于调整消费品行业商家店铺等级评价体系的公告」。

L 体系原有指标：

- GMV
- 买家数
- 旺旺响应率
- 揽收率
- 老客回头率

新增指标：

- 潜力商品数
- 采购指数4.0商品数
- 特色货盘总数
- 访客数
- 新访客数

说明：原“金冠品商品数”已随 2026-05-09 商品成长升级进入历史/过渡口径，长期目标应改用采购指数4.0商品数。

注意：不同二级类目、不同店铺等级下，同一评价指标的具体数值可能不同。因此第一版不能硬编码全部 L 等级数值，只能先做“字段跟踪 + 目标手填 + 差距诊断”。

## 4. 指标优先级

### P0：平台权益保命指标

这些指标低了，可能影响搜索推荐、活动报名、营销推广和流量权重。员工每天必须看。

- 旺旺 3 分钟响应率
- 48 小时揽收率 / 找工厂履约率
- 发货物流退款率
- 品质退款率 / 品质问题率
- 退款处理时长
- 介入率
- 咨询满意度

### P1：找工厂牌级指标

这些指标决定无牌、铜牌、银牌、金牌。

- 服务响应率
- 履约率
- 定制交易积分
- 合约支付率
- 月动销小单定制商品数

### P2：L 等级 / 店铺成长指标

这些指标决定店铺成长和后续权益。

- GMV
- 买家数
- 旺旺响应率
- 揽收率
- 老客回头率
- 潜力商品数
- 采购指数4.0商品数
- 特色货盘总数
- 访客数
- 新访客数

### P3：经营迭代指标

这些指标不是官方牌级目标，但用来解释“为什么官方指标没上去”。

- 有效询盘率
- 报价后继续沟通率
- 样品转化率
- SKU 角色清晰度
- 单品毛利率
- 复购跟进率
- SOP 沉淀数量

## 5. 数据口径和录入

第一版不假设能直接接入 1688 API，先做“半自动运营台账”。员工从后台查看数据并录入，系统负责判断和生成动作。

每个字段必须包含：

- 字段名称
- 官方规则来源
- 适用范围
- 统计周期
- 数据来源位置
- 当前值
- 目标值
- 更新时间
- 置信度
- 来源可信度

每日录入字段：

- 旺旺 3 分钟响应率
- 未及时响应消息数
- 待发货订单数
- 超承诺发货风险订单数
- 今日询盘数
- 今日报价数
- 今日合约/支付推进数
- 今日退款/售后问题数
- 今日完成任务和结果

每周录入字段：

- 近 30 天服务响应率
- 近 30 天 48 小时揽收率 / 找工厂履约率
- 定制交易积分
- 合约支付率
- 月动销小单定制商品数
- GMV、买家数、访客数、新访客数
- 老客回头率
- 品质退款率、介入率、退款处理时长
- 本周新增 SOP / 避坑记录数量

## 6. 任务生成逻辑

任务不是普通待办。每个任务都必须追溯到一个官方目标或经营目标。

任务卡字段：

- 对应指标
- 当前值
- 目标值
- 差距
- 风险等级
- 今天要做什么
- 为什么做
- 方法论卡片
- 完成后记录什么结果
- 是否沉淀成 SOP

规则示例：

| 触发条件 | 生成任务 |
| --- | --- |
| 旺旺 3 分钟响应率低 | 设置 09:00-21:00 值班、检查未响应消息、补快捷回复、检查 AI 回复有效性、当日响应复盘 |
| 48 小时揽收率 / 找工厂履约率低 | 排查风险订单、标记缺货 SKU、检查发货承诺、检查物流单号异常、确认供应链交期 |
| 定制交易积分不够 | 推广定制 SKU、推进合约订单、跟进报价后客户、样品转大货、回访定制客户 |
| 合约支付率低 | 报价单转合约、排查支付障碍、提醒付款节点、跟进未支付客户 |
| 小单定制动销商品数不足 3 款 | 补齐小单定制 SKU、前置定制能力、优化拿样政策、测试主推定制款 |
| 品质退款率 / 品质问题率高 | 建质检清单、归类差评/退款原因、反馈供应问题、暂停推广风险 SKU |
| 老客回头率低 | 建成交客户档案、7 天回访、30 天补货提醒、旺季备货提醒 |

## 7. 冲突裁判规则

官方指标之间会冲突，系统必须做裁判。

优先级规则：

1. 违规风险高于一切增长目标。
2. 毛利和现金流风险高于 GMV、交易积分和冲级目标。
3. 履约风险高于推广和接单。
4. P0 服务指标低于安全线时，暂停 P2/P3 扩张动作。
5. 找工厂冲级任务不能牺牲品质退款率、介入率和毛利。
6. 响应率提升不能靠无效自动回复或低质量敷衍回复。
7. 小单定制动销不能靠无复购、无利润、无交付把握的虚假 SKU。

系统提示示例：

- “不建议继续冲 GMV：当前毛利低于底线，先重算成本和调价。”
- “暂停推广复杂定制单：48 小时揽收或找工厂履约低于目标，先修交付。”
- “官方指标达标但经营质量差：本周重点改毛利和复购。”
- “经营动作有效但平台权益风险高：先补 3 分钟响应和履约。”

## 8. 每日执行流程

```text
打开驾驶舱
  -> 查看今日官方风险
  -> 查看今日冲刺任务
  -> 执行任务
  -> 记录结果：有效 / 无效 / 待观察
  -> 标记是否需要沉淀 SOP
```

第一屏顺序：

1. 当前官方目标：保基础服务分 / 冲铜 / 冲银 / 冲金 / 提升 L 等级 / 修复新灯塔短板。
2. 今日最高风险指标。
3. 今日 3-5 个冲刺任务。
4. 每个任务旁边展示规则口径和方法论卡片。

## 9. 每周复盘流程

```text
录入本周/近 30 天指标
  -> 系统比较目标和差距
  -> 判断进步/退步
  -> 归因到动作
  -> 生成下周目标
  -> 保存有效 SOP 和避坑记录
```

复盘输出：

- 本周进步的官方指标
- 本周退步的官方指标
- 经营底线是否被破坏
- 哪些动作可能有效
- 哪些动作无效或有副作用
- 下周只冲 1-3 个目标
- 应沉淀的 SOP
- 应避免重复的坑

## 10. 规则版本管理

官方规则会更新，系统不能把旧政策当永久真理。

每条官方规则记录：

- 规则名称
- 规则链接
- 发布时间
- 生效时间
- 适用范围
- 关键指标

## 11. 第一版页面结构

1. **目标选择**
   选择当前阶段目标：保基础服务分、冲铜牌、冲银牌、冲金牌、提升 L 等级、修复新灯塔短板。

2. **官方目标面板**
   展示 P0/P1 指标：旺旺 3 分钟响应率、48 小时揽收率、找工厂履约率、发货物流退款率、品质退款率、退款处理时长、介入率、找工厂五项指标。

3. **差距诊断**
   告诉员工离目标差多少，哪个指标风险最高，影响什么权益。

4. **今日冲刺任务**
   每个任务绑定一个官方指标或经营指标，并附带方法论卡片。

5. **每日执行记录**
   记录任务完成、结果有效/无效/待观察、是否需要沉淀 SOP。

6. **每周迭代复盘**
   总结本周哪些指标进步、哪些退步、原因是什么、下周冲什么目标。

7. **SOP / 规则库**
   存官方规则版本、链接、指标口径、对应打法、有效动作、避坑记录。

## 12. 暂不做范围

第一版暂不做：

- 自动爬取 1688 后台
- 1688 API 对接
- 多人权限
- 复杂财务系统
- 广告投放管理
- 自动判定所有 L 等级具体数值
- 库存和仓储系统
- CRM 全流程

这些不是不重要，而是会拖慢核心闭环。第一版先验证“官方目标 -> 差距 -> 动作 -> 结果 -> SOP”的迭代系统是否能让员工进步。

## 13. 关键风险

1. **规则过期风险**
   解决：所有规则都带版本和复核日期。

2. **录入错误风险**
   解决：字段旁边标注后台来源、统计周期和适用范围。

3. **指标冲突风险**
   解决：使用目标宪法和冲突裁判规则。

4. **员工执行但不复盘**
   解决：任务完成必须记录结果，周复盘必须沉淀有效动作或避坑记录。

5. **任务变成普通待办**
   解决：每个任务必须绑定指标、差距和方法论。

6. **官方指标提升但经营质量变差**
   解决：毛利、交付、违规和老板注意力作为经营底线。

## 14. 下一步

开发前先完成两件事：

1. 把官方目标字段整理成结构化数据模型。
2. 把当前已有 dashboard 重构为“目标选择 -> 官方目标面板 -> 差距诊断 -> 今日任务 -> 每周复盘”的产品结构。

第一版验收标准：

- 员工能选择一个官方目标。
- 员工能录入当前指标。
- 系统能判断目标差距。
- 系统能生成 3-5 个今日任务。
- 每个任务能说明对应指标和原因。
- 员工能记录结果并沉淀 SOP。
- 周复盘能生成下周优先目标。
