# 1688 保温杯运营助手 MVP 切片

本文档是开发前最后一刀：把已经拆清楚的产品收敛成第一版可开发、可验收、可迭代的最小闭环。

它不是完整产品蓝图。完整蓝图见：

- `docs/plans/2026-05-23-1688-official-goal-iteration-design.md`
- `docs/plans/2026-05-23-1688-page-spec.md`
- `docs/plans/2026-05-23-1688-thermos-rule-tables.md`

## 1. MVP 目标

第一版只验证一个核心假设：

> 一个“一个人全包”的保温杯 1688 运营员工，能否通过官方指标录入、系统诊断、今日任务、执行记录和周复盘，持续改进运营动作，而不是每天凭感觉忙。

MVP 闭环：

```text
选择目标
  -> 录入指标
  -> 系统诊断差距
  -> 生成今日任务
  -> 员工记录执行结果
  -> 周复盘判断有效/无效
  -> 沉淀 1 条 SOP
```

MVP 不验证：

- 自动接入 1688 后台是否可行。
- 全量 CRM 是否好用。
- 库存、财务、广告系统是否完整。
- AI 是否能自动运营。
- 多员工协作是否成立。

## 2. 第一版页面收敛

完整页面规格里有 7 个页面。MVP 先压成 3 个主页面。

| MVP 页面 | 保留内容 | 暂时吸收哪些完整页面 |
| --- | --- | --- |
| 今日冲刺 | 当前目标、最高风险、今日任务、执行记录、SOP 候选 | 询盘处理、交付售后、老客复购里的任务入口 |
| 官方目标 | 目标选择、店铺上下文、指标录入、差距诊断、阻断提示 | 规则库里的官方规则摘要 |
| 周复盘 / SOP | 本周指标变化、任务归因、下周目标、SOP 沉淀 | SOP / 规则库 |

暂不单独做页面：

- 询盘处理页
- 交付售后页
- 老客复购页
- 完整 SOP / 规则库页

这些能力在 MVP 中以“任务类型 + 方法卡 + 复盘字段”的形式存在。

## 3. MVP 用户路径

### 3.1 第一次打开

员工看到：

```text
默认店铺：保温杯消费品店铺
默认目标：保基础服务分
数据状态：待录入
今日任务：需先录入指标后生成
```

员工必须先完成：

1. 确认店铺上下文。
2. 选择当前目标。
3. 录入关键指标。
4. 生成今日任务。

### 3.2 每日使用

```text
打开今日冲刺
  -> 看最高风险
  -> 执行 3-5 个任务
  -> 记录结果：有效 / 无效 / 待观察
  -> 上传或填写完成证据
  -> 标记是否值得沉淀 SOP
```

每日结束必须留下：

- 至少 1 条任务执行记录。
- 每个完成任务的结果状态。
- 高风险任务未完成原因。

### 3.3 每周使用

```text
打开周复盘 / SOP
  -> 查看本周指标变化
  -> 查看任务有效性
  -> 判断继续 / 停止 / 换打法
  -> 选择下周 1-3 个目标
  -> 保存 1 条 SOP 或避坑记录
```

每周复盘必须输出：

- 本周进步指标。
- 本周退步指标。
- 最有效任务。
- 最无效任务。
- 下周主目标。
- 1 条 SOP 候选或避坑记录。

## 4. MVP 数据输入

第一版全部手动录入，不接 API。

### 4.1 店铺上下文

| 字段 | 默认值 | 是否必填 |
| --- | --- | --- |
| 主类目 | 保温杯 | 是 |
| 类目类型 | 消费品 / 日用百货 | 是 |
| 主订单类型 | 混合 | 是 |
| 是否入驻找工厂 | 未确认 | 是 |
| 当前找工厂牌级 | 无牌 / 铜 / 银 / 金 / 未确认 | 是 |
| 当前 L 等级 | 手填 | 否 |
| 毛利底线 | 手填百分比 | 是 |
| 老板日常介入上限 | 默认 0.5 小时/天 | 否 |

### 4.2 每日指标

| 指标 | 默认目标 | 用途 |
| --- | ---: | --- |
| 旺旺 3 分钟响应率 | 60% | P0 咨询体验、冲铜服务响应底线参考 |
| 未及时响应消息数 | 0 | 解释响应率问题 |
| 待发货订单数 | 0 | 判断交付压力 |
| 48 小时揽收风险订单数 | 0 | 保护消费品物流体验 |
| 今日询盘数 | 手填 | 识别流量和客服压力 |
| 今日报价数 | 手填 | 识别报价效率 |
| 今日退款/售后数 | 0 | 识别售后压力 |
| 今日毛利异常订单数 | 0 | 阻断盲目增长 |

### 4.3 每周指标

| 指标 | 默认目标 | 用途 |
| --- | ---: | --- |
| 近 30 天服务响应率 | 铜牌 60% / 银牌 65% / 金牌 70% | 找工厂牌级 |
| 近 30 天 48 小时揽收率 | 手填 | 消费品物流体验 |
| 找工厂履约率 | 铜牌 70% / 银牌 75% / 金牌 80% | 找工厂牌级 |
| 定制交易积分 | 铜 10 万 / 银 80 万 / 金 150 万 | 找工厂牌级 |
| 合约支付率 | 铜 50% / 银 60% / 金 70% | 找工厂牌级 |
| 月动销小单定制商品数 | 3 款 | 找工厂牌级 |
| 品质退款率 | 手填 | 商品体验 |
| 退款处理时长 | 手填 | 售后体验 |
| 平台介入率 | 手填 | 售后风险 |
| 老客回头率 | 手填 | 经营稳定性 |
| 本周 SOP 新增数 | ≥1 | 迭代资产 |

## 5. MVP 种子场景

第一版先内置 5 个典型场景，用来验证诊断和任务生成是否成立。

### S1：响应率低

输入：

```text
当前目标：冲找工厂铜牌
旺旺 3 分钟响应率：52%
近 30 天服务响应率：55%
未及时响应消息数：20
```

诊断：

```text
P0 咨询体验风险；P1 找工厂服务响应率未达铜牌底线。
先修客服响应，不优先做商品标题优化。
```

生成任务：

- 检查 20 条未响应消息原因。
- 补 20 条保温杯快捷回复。
- 设置 09:00-21:00 值班提醒。

完成后记录：

- 当日响应率。
- 未响应原因分类。
- 快捷回复是否沉淀 SOP。

### S2：48 小时揽收风险

输入：

```text
当前目标：保基础服务分
48 小时揽收风险订单数：8
待发货订单数：23
今日退款/售后数：3
```

诊断：

```text
P0 物流体验风险。保温杯现货和样品单必须先保护发货，不建议今天扩张推广。
```

生成任务：

- 排查 48 小时内需揽收订单。
- 标记缺货、待确认、物流异常。
- 检查商品发货承诺是否过度。

完成后记录：

- 已处理风险订单数。
- 未处理原因。
- 是否需要修改发货日清 SOP。

### S3：小单定制不足

输入：

```text
当前目标：冲找工厂铜牌
月动销小单定制商品数：1
定制交易积分：60000
合约支付率：45%
```

诊断：

```text
找工厂冲级入口不足。至少要有 3 个可成交的小单定制入口。
```

生成任务：

- 选择 3 个适合刻字/印 LOGO 的保温杯款。
- 补“支持拿样、可刻字、可印 LOGO、MOQ、打样周期”信息。
- 跟进高意向定制询盘。

完成后记录：

- 商品链接。
- 今日定制询盘数。
- 是否产生报价或支付推进。

### S4：品质退款上升

输入：

```text
当前目标：提升 L 等级
品质退款率：高于内部警戒线
今日退款/售后数：6
涉及 SKU：2 个主推保温杯款
```

诊断：

```text
商品体验风险。不能继续扩大主推款流量，先判断是产品真问题还是详情页预期管理问题。
```

生成任务：

- 按漏水、保温、掉漆、容量、包装、错发分类售后原因。
- 暂停推广高风险 SKU。
- 补详情页风险说明或反馈供应链。

完成后记录：

- 问题 SKU。
- 问题类型。
- 停推/补说明/换供应链动作。

### S5：GMV 增长但毛利低

输入：

```text
当前目标：提升 L 等级
GMV：上升
毛利率：低于毛利底线
今日毛利异常订单数：5
```

诊断：

```text
增长质量不合格。不能为了 L 等级或交易积分牺牲经营底线。
```

生成任务：

- 补单品毛利表。
- 拆产品、包装、运费、样品、售后、广告成本。
- 标记低毛利订单来源。

完成后记录：

- 单品毛利率。
- 低毛利原因。
- 调价或停推建议。

## 6. MVP 诊断输出格式

每次录入指标后，系统输出 4 类结果。

### 6.1 风险摘要

```text
最高风险：旺旺 3 分钟响应率
风险等级：P0
当前值：52%
目标值：60%
差距：-8%
影响：咨询体验、找工厂服务响应、搜索推荐、活动权益
```

### 6.2 阻断提示

```text
响应率未回到目标前，不优先做商品标题优化。
```

或：

```text
毛利低于底线，不建议继续冲 GMV 或定制交易积分。
```

### 6.3 今日任务

```text
任务标题：补齐 09:00-21:00 客服响应机制
优先级：P0
对应指标：旺旺 3 分钟响应率、找工厂服务响应率
触发规则：R01、R02
今天动作：检查未响应消息、补快捷回复、设置值班提醒
完成证据：未响应原因清单、快捷回复截图、当日响应率
```

### 6.4 周复盘问题

```text
响应慢是人手问题、提醒问题、话术问题，还是低质量询盘过多？
```

## 7. MVP 不做范围

第一版明确不做：

- 独立询盘处理完整页面。
- 独立交付售后完整页面。
- 独立老客复购完整页面。
- 自动读取 1688 后台。
- 自动聊天、自动报价、自动发货。
- 多员工权限和绩效管理。
- 广告投放管理。
- 库存和仓储系统。
- 完整 CRM。
- 全量 L 等级目标自动计算。
- 图片上传、截图 OCR、浏览器插件。

这些都先不做，除非 MVP 闭环无法验证。

## 8. MVP 页面验收

### 今日冲刺

验收标准：

- 员工打开后 30 秒内能看到今日最高风险。
- 今日任务最多 5 个，且按 P0/P1/P2/P3 排序。
- 每个任务都有对应指标、触发规则、今天动作、完成证据。
- 任务可以记录：未开始 / 进行中 / 已完成 / 被阻断。
- 任务结果必须选择：有效 / 无效 / 待观察。
- 可以把任务标记为 SOP 候选。

### 官方目标

验收标准：

- 可以选择：保基础服务分、冲铜、冲银、冲金、提升 L 等级、修复新灯塔短板。
- 可以录入店铺上下文和关键指标。
- 可以看到找工厂铜/银/金五项差距。
- 可以看到 P0 服务指标风险。
- 未入驻找工厂时，冲牌级目标显示入场资格不足。
- 毛利低于底线时，增长目标被阻断。

### 周复盘 / SOP

验收标准：

- 可以看到本周进步、退步、无变化但高风险的指标。
- 可以看到本周任务的有效 / 无效 / 待观察分布。
- 可以选择下周 1-3 个主目标。
- 可以从 SOP 候选生成一条 SOP。
- 可以记录一个“停止动作”。
- 可以记录 Leon 是否被拖入日常。

## 9. 开发数据边界

MVP 可以先使用本地静态种子数据，不需要真实账号。

建议内置：

- 1 个默认店铺上下文：保温杯消费品店铺。
- 6 个目标预设。
- 15-20 个指标定义。
- 5 个种子场景。
- 10-15 个任务模板。
- 3 条示例执行记录。
- 1 条示例周复盘。
- 1 条示例 SOP。

所有数据结构必须保留未来接 API 的空间：

- `source`
- `sourceUrl`
- `ruleVersion`
- `lastCheckedAt`
- `confirmedByUser`
- `confidence`

## 10. 进入开发的判断

满足以下条件后，可以开始开发：

- MVP 只做 3 个主页面。
- 种子场景只做 5 个。
- 第一版只做手动录入。
- 任务生成只覆盖 R01、R02、R03、R04、R06、R09、R15、R18、R20。
- 不做完整 CRM、库存、广告、API。
- 验收只看“员工是否能完成每日执行 + 周复盘 + SOP 沉淀”。

开发第一步应该是：

```text
先实现诊断和任务生成的纯函数，再做页面。
```

原因：

- 页面只是展示。
- 产品价值来自“指标 -> 诊断 -> 任务 -> 复盘”的因果链。
- 纯函数最容易测试，能防止 UI 先行导致规则混乱。

## 11. 后续扩展顺序

MVP 跑通后再按这个顺序扩展：

1. 独立询盘处理页。
2. 独立交付售后页。
3. 独立老客复购页。
4. 完整 SOP / 规则库。
5. 指标导入或截图辅助录入。
6. 1688 后台/API/插件能力。
7. 多员工协作和管理视图。

不要反过来先做管理后台。
