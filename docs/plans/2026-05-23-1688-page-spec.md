# 1688 保温杯运营助手第一版页面规格

本文档把规则层拆成第一版可用页面。它不是 UI 视觉稿，也不是开发任务清单；它定义每个页面要帮员工做什么判断、录入什么数据、触发什么任务、沉淀什么资产。

第一版默认：

- 主类目：保温杯
- 类目类型：消费品 / 日用百货
- 使用者：一个人全包的运营员工
- 经营形态：现货批发 + 拿样 + LOGO 定制 + 礼品团购 + 老客补货
- 工作节奏：每日执行 + 每周复盘

## 1. 页面总原则

第一版页面不追求“大屏感”，而追求让员工每天少犯错、知道先做什么、做完能复盘。

页面优先级：

```text
目标
  -> 风险
  -> 任务
  -> 执行证据
  -> 复盘
  -> SOP
```

关键原则：

- 每个任务必须绑定指标或经营底线。
- P0 风险必须在首屏出现。
- 官方指标和经营指标要分开显示。
- 员工不能只勾选完成，必须记录结果和证据。
- 页面要帮助员工判断“继续、停止、换打法”。
- 找工厂牌级是官方成长目标，不是保温杯经营成功本身。

## 2. 全局导航

第一版建议用顶部导航 + 页面内标签，不做复杂后台。

顶部导航：

1. 今日冲刺
2. 官方目标
3. 询盘处理
4. 交付售后
5. 老客复购
6. 周复盘
7. SOP / 规则库

导航展示规则：

| 导航项 | 什么时候出现红点 | 点击后主要解决什么 |
| --- | --- | --- |
| 今日冲刺 | 有 P0/P1 高风险任务未处理 | 今天先做什么 |
| 官方目标 | 目标被阻断、规则待确认、指标缺失 | 为什么做、差多少、能不能冲 |
| 询盘处理 | 有未分类询盘、报价后未跟进 | 询盘转化 |
| 交付售后 | 有待发货风险、退款、品质问题 | 发货和售后风险 |
| 老客复购 | 有到期回访、老客信息缺失 | 复购和客户资产 |
| 周复盘 | 到周复盘日、连续任务无效果 | 迭代进步 |
| SOP / 规则库 | 有待沉淀 SOP、规则可能过期 | 资产沉淀和规则复核 |

## 3. 今日冲刺页

### 3.1 页面目标

让员工一打开软件就知道：

1. 当前冲什么目标。
2. 今天哪个指标最危险。
3. 今天只做哪 3-5 件事。
4. 做完要记录什么证据。

### 3.2 首屏布局

```text
当前目标条
  - 当前目标：保服务分 / 冲铜 / 冲银 / 冲金 / 提升 L 等级 / 修复新灯塔
  - 状态：可冲 / 被阻断 / 风险高 / 数据不足
  - 周期：本周第几天 / 本轮目标剩余几天

最高风险卡
  - 风险指标
  - 当前值 / 目标值 / 差距
  - 影响权益
  - 阻断动作

今日任务队列
  - P0/P1/P2/P3 标签
  - 任务标题
  - 对应指标
  - 今天动作
  - 完成证据

任务详情侧栏
  - 官方规则口径
  - 方法论卡片
  - 注意事项
  - 周复盘问题
```

### 3.3 页面字段

| 区域 | 字段 | 来源 |
| --- | --- | --- |
| 当前目标 | `selectedGoalId`、目标名称、目标状态、周期 | GoalPreset、IterationCycle |
| 最高风险 | 指标名、当前值、目标值、差距、风险等级 | DiagnosticGap |
| 任务队列 | 任务标题、优先级、触发规则、对应指标、截止时间 | TaskInstance |
| 任务详情 | 今天动作、为什么做、完成证据、阻断规则 | TaskTemplate、RuleTable |
| 执行记录 | 完成状态、结果、证据、备注、是否沉淀 SOP | ExecutionLog |

### 3.4 关键操作

| 操作 | 员工动作 | 系统反应 |
| --- | --- | --- |
| 选择今日主目标 | 从系统建议中选 1 个 | 其他任务降级为辅助任务 |
| 开始任务 | 点击任务卡 | 展开动作步骤和完成证据 |
| 记录结果 | 选有效 / 无效 / 待观察 | 写入执行日志，等待周复盘归因 |
| 标记 SOP | 勾选“值得沉淀” | 周复盘页生成 SOP 候选 |
| 阻断任务 | 选择“今天不能做”并填写原因 | 生成阻断记录，周复盘必须解释 |

### 3.5 任务卡标准

```text
优先级：
任务标题：
对应指标：
当前值：
目标值：
差距：
触发规则：
订单类型：
今天动作：
完成证据：
阻断规则：
复盘问题：
```

### 3.6 今日冲刺示例

```text
P0 | 补齐 09:00-21:00 客服响应机制
对应指标：旺旺 3 分钟响应率、找工厂服务响应率
当前值：52% / 目标：60%
触发规则：R01、R02
今天动作：检查 20 条未响应消息，补 20 条保温杯快捷回复，开启手机提醒
完成证据：未响应原因清单、快捷回复截图、当日响应率
阻断规则：响应率未回到目标前，不优先做商品标题优化
```

## 4. 官方目标页

### 4.1 页面目标

让员工知道目标从哪里来、当前差距是什么、能不能冲、被什么经营底线阻断。

### 4.2 页面结构

```text
目标选择区
  - 保基础服务分
  - 冲找工厂铜牌
  - 冲找工厂银牌
  - 冲找工厂金牌
  - 提升 L 等级
  - 修复新灯塔短板

店铺上下文区
  - 主类目：保温杯
  - 类目类型：消费品 / 混合 / 其他
  - 主订单类型：现货 / 定制 / 合约 / 邀约 / 混合
  - 是否入驻找工厂
  - 当前牌级
  - 当前 L 等级
  - 毛利底线

指标差距区
  - P0 平台保命
  - P1 找工厂牌级
  - P2 L 等级成长
  - P3 经营迭代

规则来源区
  - 官方规则链接
  - 规则版本
  - 最近复核时间
  - 状态
```

### 4.3 目标选择逻辑

| 目标 | 入场资格 | 必看指标 | 可生成任务 |
| --- | --- | --- | --- |
| 保基础服务分 | 默认可选 | 3 分钟响应、48 小时揽收、退款、品质、介入 | R01、R03、R05-R08 |
| 冲找工厂铜牌 | 已入驻找工厂，非超级工厂 | 服务响应率、履约率、定制交易积分、合约支付率、小单定制动销 | R02、R04、R09-R11 |
| 冲找工厂银牌 | 已达或接近铜牌基础 | 同上，目标值更高 | R02、R04、R09-R11 |
| 冲找工厂金牌 | 已达或接近银牌基础 | 同上，目标值最高 | R02、R04、R09-R11 |
| 提升 L 等级 | 有类目目标值或手填目标 | GMV、买家数、旺旺响应率、揽收率、老客回头率、潜力商品等 | R12、R16、R17 |
| 修复新灯塔短板 | 任一服务体验短板明显 | 物流、咨询、商品、售后体验 | R01、R03、R05-R08 |

### 4.4 阻断提示

系统必须明确告诉员工“为什么现在不建议冲”。

阻断示例：

| 阻断原因 | 页面提示 |
| --- | --- |
| 未入驻找工厂 | 当前无法冲找工厂牌级，先确认入驻资格 |
| 毛利低于底线 | 不建议冲 GMV 或交易积分，先重算报价和成本 |
| 48 小时揽收风险高 | 暂停推广复杂定制和大促动作，先修交付 |
| 品质退款率上升 | 不建议扩大主推款流量，先排查质量问题 |
| 规则未确认 | 当前规则可能过期，先人工复核官方来源 |

## 5. 询盘处理页

### 5.1 页面目标

把“聊天”变成可复盘的成交流程。员工必须先识别询盘类型，再按类型处理，而不是所有客户都直接报价。

### 5.2 页面结构

```text
未处理询盘列表
  - 买家
  - 询盘来源商品
  - 首次消息时间
  - 是否超过 3 分钟
  - 当前状态

询盘分类面板
  - 现货批发
  - 拿样试单
  - LOGO 定制
  - 礼品团购
  - 老客补货
  - 渠道分销
  - 低质量询盘

四问法面板
  - 用途
  - 数量
  - 预算
  - 交期

报价推进面板
  - 三档报价
  - 样品政策
  - 含税/含运
  - 打样/大货周期
  - 下次跟进时间
```

### 5.3 询盘分类后的动作

| 询盘类型 | 默认动作 | 必填字段 | 下一步 |
| --- | --- | --- | --- |
| 现货批发 | 报阶梯价、确认库存、确认发货 | 数量、规格、交期、是否开票 | 报价后跟进 |
| 拿样试单 | 报样品价和样品发货时间 | 样品款、潜在大货数量、回访时间 | 3/7 天回访 |
| LOGO 定制 | 问工艺、数量、预算、交期 | LOGO、印刷方式、MOQ、打样周期 | 生成定制报价 |
| 礼品团购 | 给低中高三档方案 | 节日节点、包装、发票、对公 | 生成方案报价 |
| 老客补货 | 调历史订单和规格 | 历史规格、历史价格、补货周期 | 复用报价 |
| 渠道分销 | 建分销客户档案 | 渠道类型、目标利润、售后边界 | 进入客户资产 |
| 低质量询盘 | 快速筛选并归档 | 缺失原因、来源商品 | 低投入跟进 |

### 5.4 询盘页关键指标

- 3 分钟响应率
- 有效询盘率
- 报价后继续沟通率
- 报价后成交率
- 样品转大货率
- 低质量询盘占比

## 6. 交付售后页

### 6.1 页面目标

保护 48 小时揽收、履约、退款、品质和介入率。这个页面不是订单系统，而是风险处理台。

### 6.2 页面结构

```text
发货风险
  - 48 小时内需揽收订单
  - 缺货风险
  - 待客户确认
  - 物流异常

定制确认
  - 材质
  - 容量
  - 颜色
  - 印刷
  - 包装
  - 交期
  - 确认稿

售后风险
  - 待处理退款
  - 品质退款
  - 物流退款
  - 可能介入订单

问题归因
  - SKU
  - 供应链
  - 物流
  - 客服承诺
  - 详情页预期
```

### 6.3 风险处理规则

| 风险 | 默认任务 | 记录证据 | 可能阻断 |
| --- | --- | --- | --- |
| 待发货超时 | 发货日清、确认库存和物流 | 风险订单清单 | 暂停推广现货款 |
| 定制确认缺失 | 补确认稿和交期确认 | 确认稿、聊天记录 | 不推进大货 |
| 品质退款上升 | 分类质量问题 | 售后原因表 | 停推问题 SKU |
| 物流退款上升 | 查物流商、发货承诺、包装 | 物流异常表 | 更换发货方案 |
| 介入风险 | 复盘承诺和证据 | 介入原因记录 | 升级负责人 |

## 7. 老客复购页

### 7.1 页面目标

保温杯店铺要从“每次重新找客户”变成“客户资产可复用”。这个页面管理采购周期、历史规格、回访和复购机会。

### 7.2 页面结构

```text
今日回访
  - 成交后 7 天反馈
  - 成交后 30 天补货
  - 旺季前备货提醒

客户档案
  - 买家类型
  - 常买规格
  - 历史价格
  - 开票需求
  - 交期偏好
  - 售后记录

复购机会
  - 到期补货
  - 新品推荐
  - 礼品节点
  - 规格扩展
```

### 7.3 客户分层

| 客户层级 | 特征 | 默认动作 |
| --- | --- | --- |
| A 类老客 | 已复购、有稳定规格、售后少 | 固定周期回访，优先新品和备货提醒 |
| B 类机会客户 | 成交过一次，反馈正常 | 7/30 天回访，问补货和新需求 |
| C 类样品客户 | 拿样未下大货 | 3/7 天跟进样品反馈 |
| D 类风险客户 | 售后未解决或利润低 | 先解决问题，不强推复购 |

### 7.4 复购页关键指标

- 老客回头率
- 复购机会数
- 回访完成率
- 样品转大货率
- 老客成交毛利率

## 8. 周复盘页

### 8.1 页面目标

周复盘不是写周报，而是判断“本周动作有没有让员工和店铺变好”。

### 8.2 页面结构

```text
本周指标变化
  - 进步指标
  - 退步指标
  - 未变化但仍高风险指标

任务归因
  - 有效任务
  - 无效任务
  - 有副作用任务
  - 未完成任务

经营底线
  - 毛利是否低于底线
  - 交付是否被伤害
  - 售后是否吞利润
  - Leon 是否被拖入日常

下周目标
  - 继续
  - 停止
  - 换打法
  - 下周 1-3 个主目标

SOP 沉淀
  - 有效打法
  - 避坑记录
  - 待验证打法
```

### 8.3 周复盘必须回答

1. 哪个官方指标变好了。
2. 哪个官方指标变差了。
3. 哪个经营底线被伤害了。
4. 哪个任务可能有效。
5. 哪个任务只是忙。
6. 哪个动作下周要停止。
7. 哪个经验要沉淀成 SOP。
8. 下周只冲哪 1-3 个目标。

### 8.4 下周目标生成规则

| 本周情况 | 下周建议 |
| --- | --- |
| P0 指标未达标 | 下周继续修 P0，不扩张 |
| P0 达标但找工厂指标差 | 选择 1 个找工厂短板作为主目标 |
| 找工厂指标改善但毛利差 | 停止冲规模，修报价和成本 |
| 询盘增加但成交没变 | 修询盘分类和报价模板 |
| 成交增加但售后上升 | 修交付和品质，不继续放大 |
| 任务多但指标没动 | 停止低效任务，重拆目标 |

## 9. SOP / 规则库页

### 9.1 页面目标

把有效动作沉淀成资产，把官方规则和内部方法放在同一个可追溯系统里。

### 9.2 页面结构

```text
官方规则库
  - 找工厂牌级规则
  - 商家服务管理规范
  - 消费品 L 等级规则
  - 规则状态

方法论库
  - 客服四问法
  - 三档报价
  - 标题关键词矩阵
  - 定制确认 SOP
  - 老客回访 SOP

店铺 SOP
  - 已验证 SOP
  - 待验证 SOP
  - 避坑记录
```

### 9.3 SOP 卡字段

| 字段 | 说明 |
| --- | --- |
| 标题 | 例如“3 分钟响应值班 SOP” |
| 适用场景 | 哪类指标、订单类型或问题 |
| 触发条件 | 什么时候使用 |
| 操作步骤 | 员工照着做的步骤 |
| 证据要求 | 做完要留什么 |
| 关联指标 | 能影响哪些指标 |
| 上次使用结果 | 有效 / 无效 / 待观察 |
| 置信度 | low / medium / high |
| 最后使用时间 | 用于淘汰过期 SOP |

## 10. 空状态

第一版必须把空状态做好，否则员工第一次打开不知道填什么。

| 空状态 | 页面提示 | 默认动作 |
| --- | --- | --- |
| 未选择目标 | 先从“保基础服务分”开始 | 进入官方目标页选择目标 |
| 未填写店铺上下文 | 先确认主类目、订单类型、是否找工厂 | 打开店铺上下文表单 |
| 无指标读数 | 暂时无法生成差距诊断 | 展示后台数据来源和录入示例 |
| 无今日任务 | 当前没有高风险任务 | 建议沉淀 SOP 或做周复盘 |
| 无询盘 | 今天无询盘可处理 | 去商品页检查主推款、标题、访客 |
| 无老客 | 尚未形成客户资产 | 从已成交订单建立第一批客户档案 |
| 无 SOP | 还没有可复用经验 | 从本周有效任务生成第一条 SOP |

## 11. 第一版不做

第一版暂不做：

- 自动接入 1688 后台。
- 多员工权限。
- 财务系统。
- 库存系统。
- 广告投放管理。
- 全量 CRM。
- 自动判定所有 L 等级具体数值。
- 自动聊天或自动报价。

这些能力后面可以做，但第一版先验证：员工能否通过官方指标、保温杯动作库和周复盘持续迭代进步。

## 12. 第一版验收标准

产品方案层面的验收：

- 员工打开“今日冲刺”能在 30 秒内知道今天最重要的 1 个目标。
- 每个任务都能追溯到一个指标、规则或经营底线。
- 每个任务都有完成证据，而不是只勾选完成。
- 官方目标页能说明找工厂铜/银/金差距和阻断原因。
- 询盘处理页能区分现货、拿样、LOGO 定制、礼品团购、老客补货和低质量询盘。
- 交付售后页能保护 48 小时揽收、品质退款、退款处理和介入率。
- 老客复购页能沉淀客户规格、周期和回访任务。
- 周复盘页能输出“继续、停止、换打法”和 1 条 SOP 候选。
- SOP / 规则库能区分官方规则、方法论和本店验证过的 SOP。
