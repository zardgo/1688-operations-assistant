# 1688 运营助手 V4：每日经营数据与目标扩展

## 背景

V3 已经形成“官方目标 + 经营目标 + 能力目标”的经营推理层，但数据录入仍偏少。用户提供的代运营公司统计表显示，真实运营现场会按天追踪曝光、广告、访客、询盘、支付、商品资产和平台服务风险。

V4 不应照抄代运营表，而应吸收它的每日数据颗粒度，并补上它缺失的利润、转化率、归因、动作实验和员工能力训练。

## V4 产品方向

### 一句话定义

V4 要把当前系统从“目标推理工作台”升级为“每日经营事实表 + 自动转化计算 + 官方目标侧栏 + 异常归因实验”的经营系统。

### 从代运营表借鉴什么

| 借鉴项 | V4 吸收方式 | 注意事项 |
| --- | --- | --- |
| 每日记录 | 建立每日经营数据录入页 | 保留日维度，但不只看绝对数 |
| 总曝光、广告曝光、自然曝光 | 建立流量结构目标 | 必须计算自然占比和广告依赖 |
| 广告消耗 | 建立投放效率目标 | 必须关联询盘、支付和毛利 |
| 访客、询盘、支付 | 建立漏斗转化目标 | 必须自动计算转化率 |
| 店铺层级排名 | 建立竞争位置观察项 | 不作为单独核心目标，只作趋势信号 |
| 现货商品数、潜力品数、金冠品数 | 建立商品资产目标 | 必须结合 SKU 角色，不追求无效铺货 |
| 右侧官方风险侧栏 | 做成官方目标侧栏 | 与 V3 的三层目标打通 |

### 代运营表不能直接照搬的部分

1. 没有利润，容易把“数据增长”误判为“经营变好”。
2. 没有转化率，访客、询盘、支付只是原始计数。
3. 没有原因归因，无法解释为什么某天数据异常。
4. 没有动作记录，无法知道哪个动作有效。
5. 没有 SKU 级角色，商品数增长可能只是无效铺货。
6. 没有员工能力复盘，不能训练员工判断力。

## V4 模块设计

### 1. 每日经营数据表

新增每日录入字段：

| 字段 | 频率 | 用途 |
| --- | --- | --- |
| 总曝光 | 每日 | 判断整体流量规模 |
| 广告曝光 | 每日 | 判断付费流量占比 |
| 自然曝光 | 每日 | 判断自然搜索能力 |
| 广告消耗 | 每日 | 判断投放成本 |
| 访客 | 每日 | 判断曝光到访问的吸引力 |
| 询盘 | 每日 | 判断访问后的咨询意图 |
| 支付笔数 | 每日 | 判断成交闭环 |
| 支付金额 | 每日 | 判断交易规模 |
| 店铺层级排名 | 每日/每周 | 判断平台竞争位置 |
| 现货商品数 | 每周 | 判断可承接现货供给 |
| 潜力品数 | 每周 | 判断商品成长资产 |
| 金冠品数 | 每周 | 判断优质商品资产 |
| 补单买家数 | 每日/每周 | 判断新增成交客户 |

### 2. 自动计算指标

V4 不让员工只看原始数据，必须自动计算：

| 指标 | 公式 | 目标含义 |
| --- | --- | --- |
| 自然曝光占比 | 自然曝光 / 总曝光 | 降低广告依赖 |
| 广告曝光占比 | 广告曝光 / 总曝光 | 判断付费依赖风险 |
| 曝光访客率 | 访客 / 总曝光 | 判断标题、主图、商品吸引力 |
| 访客询盘率 | 询盘 / 访客 | 判断商品详情和价格承接 |
| 询盘支付率 | 支付笔数 / 询盘 | 判断客服、报价、信任和交付承诺 |
| 单询盘广告成本 | 广告消耗 / 询盘 | 判断广告获客效率 |
| 单支付广告成本 | 广告消耗 / 支付笔数 | 判断投放是否能成交 |
| 支付客单价 | 支付金额 / 支付笔数 | 判断客户质量和 SKU 结构 |
| 广告费率 | 广告消耗 / 支付金额 | 判断广告是否吞利润 |

### 3. 官方目标侧栏

参考代运营表右侧，但做成结构化目标：

| 目标组 | 可记录指标 |
| --- | --- |
| 店铺等级 | 当前 L 等级、距离下一等级积分、缺口 |
| 交易勋章 | 当前等级、近 30 天交易额、下一等级缺口 |
| 服务体验星级 | 当前星级、目标星级、扣分项 |
| 响应体验 | 旺旺 3 分钟响应率、咨询满意度 |
| 品质体验 | 近 30 天品质退款率、品质问题率 |
| 物流体验 | 48 小时揽收率、发货/退款时效 |
| 找工厂 | 牌级、服务响应率、履约率、定制积分、合约支付率、小单定制动销商品数 |

### 4. 异常归因与动作实验

每个异常不能只显示红色，必须进入 V3 实验卡：

```text
异常：访客询盘率下降
可能原因：
1. 主图吸引了错误客户
2. 价格、起订量、定制说明不清楚
3. 详情页没有证明工厂可信
今日实验：
重写 3 个主推款详情页首屏，前置 MOQ、拿样、定制、交期、开票
预期：
3 天内访客询盘率回升
停止条件：
曝光和访客不变，询盘率仍无改善，则改查商品定位和关键词
```

## V4 优先级建议

V4 不要一次性做完全部指标。建议分两轮：

### V4.1：每日经营事实表

先做：

1. 总曝光、广告曝光、自然曝光、广告消耗、访客、询盘、支付笔数、支付金额。
2. 自动计算自然曝光占比、曝光访客率、访客询盘率、询盘支付率、单询盘广告成本、广告费率。
3. 把异常接入 V3 原因假设和实验卡。

### V4.2：商品资产与官方侧栏

再做：

1. 店铺 L 等级、交易勋章、服务体验星级、48 小时揽收率等官方侧栏。
2. 现货商品数、潜力品数、金冠品数、补单买家数。
3. SKU 角色目标：引流款、利润款、定制款、复购款、风险款、淘汰款。

## 关键产品原则

V4 的核心不是“录更多数据”，而是让员工每天能回答：

1. 今天流量是自然增长，还是广告堆出来的？
2. 访客为什么没有变成询盘？
3. 询盘为什么没有变成支付？
4. 哪些 SKU 带来有效询盘和利润？
5. 哪个动作真的改善了数据？
6. 哪个动作应该停止？
7. 今天留下了什么客户、SKU、关键词、话术、SOP 或利润表资产？
