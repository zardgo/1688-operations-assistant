# 1688 官方目标迭代驾驶舱页面信息架构

## 1. 信息架构原则

这个产品的第一屏要回答员工的三个问题：

1. 我现在冲什么目标？
2. 今天哪个指标最危险？
3. 我今天具体做哪几件事，做完记录什么结果？

不是先展示大屏报表，也不是先展示方法论文章。

页面优先级：

```text
目标
  -> 风险
  -> 任务
  -> 记录
  -> 复盘
  -> SOP
```

## 2. 推荐导航结构

第一版建议做单页应用，用顶部模式切换，不做复杂侧边栏。

顶部导航：

1. **今日冲刺**
2. **官方目标**
3. **周复盘**
4. **SOP / 规则库**

原因：

- 员工每天主要进入“今日冲刺”。
- 官方目标是当前任务的依据。
- 周复盘是迭代闭环。
- SOP/规则库是沉淀区，不应该抢首屏。

## 3. 今日冲刺页

目标：让员工一打开就知道今天做什么。

页面结构：

```text
当前目标条
  - 当前目标：冲铜 / 冲银 / 保服务分 / 修复新灯塔
  - 当前状态：可冲 / 被阻断 / 风险高
  - 最高风险：例如 旺旺3分钟响应率 52% / 目标 60%

今日任务队列
  - 3-5 个任务
  - 每个任务绑定指标、目标值、当前值、差距
  - 每个任务有“为什么做”和“完成后记录什么”

任务旁方法卡
  - 官方规则口径
  - 对应打法
  - 注意事项

执行记录
  - 完成状态
  - 结果：有效 / 无效 / 待观察
  - 证据
  - 是否沉淀 SOP
```

任务卡示例：

```text
P0 | 补 09:00-21:00 客服值班和 3 分钟响应检查
对应指标：旺旺 3 分钟响应率
当前值：52% / 目标：60%
为什么做：低于冲铜牌服务响应底线，且影响咨询体验和平台权益
今天动作：检查未响应消息、补 20 条快捷回复、确认 AI 回复有效
完成后记录：当日 3 分钟响应率、未及时响应消息数、无效回复原因
```

## 4. 官方目标页

目标：让员工知道目标从哪里来、差距在哪里、是否有资格冲。

当前第一版默认店铺类目为 **保温杯**，按消费品/日用百货处理。页面可以保留类目字段，但默认值应是保温杯消费品店铺，其他类目只作为未来扩展。

页面结构：

```text
目标选择
  - 保基础服务分
  - 冲找工厂铜牌
  - 冲找工厂银牌
  - 冲找工厂金牌
  - 提升 L 等级
  - 修复新灯塔短板

店铺上下文
  - 主类目：保温杯
  - 类目类型：消费品 / 混合 / 其他
  - 主订单类型：现货 / 定制 / 合约 / 邀约 / 混合
  - 是否入驻找工厂
  - 当前牌级
  - 当前 L 等级

指标矩阵
  - P0 平台保命
  - P1 找工厂牌级
  - P2 L 等级成长
  - P3 经营迭代

规则来源
  - 规则名称
  - 发布时间
  - 生效时间
  - 状态：待确认 / 已确认 / 可能过期
```

关键交互：

- 选择“冲金牌”时，如果当前未入驻找工厂，显示“入场资格不足，先完成找工厂入驻确认”。
- 选择“提升 L 等级”时，允许用户手填类目目标值，因为不同类目数值不同。
- 指标旁边必须显示数据口径和来源提示。

## 5. 周复盘页

目标：让员工每周看到自己是否进步，并决定下周只改 1-3 个目标。

页面结构：

```text
本周指标录入
  - 官方指标
  - 经营底线
  - 经营迭代指标

本周变化
  - 进步指标
  - 退步指标
  - 未变化但仍高风险指标

动作归因
  - 哪些任务可能有效
  - 哪些任务无效
  - 哪些任务有副作用

下周目标建议
  - 系统建议 1-3 个目标
  - 被阻断目标
  - 暂停动作

SOP 沉淀
  - 保存有效打法
  - 保存避坑记录
```

周复盘输出示例：

```text
本周响应率从 52% 到 61%，已达到冲铜服务响应底线。
可能有效动作：09:00-21:00 值班、未响应消息排查、快捷回复补充。
下周不要扩张 GMV：48 小时揽收或找工厂履约仍低于目标，先修发货和供应链确认。
建议沉淀 SOP：3 分钟响应值班清单。
```

## 6. SOP / 规则库页

目标：让员工能复用有效动作，而不是每周重新摸索。

页面结构：

```text
规则库
  - 找工厂牌级规则
  - 商家服务管理规范
  - L 等级/店铺成长规则

方法论库
  - 标题矩阵
  - 客服四问法
  - 三档报价
  - 交付确认
  - 老客复购

店铺 SOP
  - 有效 SOP
  - 避坑记录
  - 待验证打法
```

SOP 卡字段：

- 标题
- 适用指标
- 适用场景
- 操作步骤
- 证据
- 置信度
- 最后使用时间

## 7. 关键空状态

必须设计空状态，否则新员工第一次打开会不知道填什么。

空状态：

- 未选择目标：引导先选“保基础服务分”。
- 未填写店铺上下文：提示先确认类目、订单类型、是否找工厂。
- 无指标读数：展示字段来源和示例值。
- 无任务：提示当前指标达标，建议沉淀 SOP 或做周复盘。
- 无 SOP：提示从本周有效任务生成第一条 SOP。

## 8. 视觉和交互重点

这个产品是运营工具，不是营销页。

设计要求：

- 信息密度要高，但层级清楚。
- P0 风险必须一眼可见。
- 官方指标和经营指标要视觉区分。
- 每个任务都要显示“对应指标”。
- 每个指标都要显示“数据来源/统计周期”。
- 不用大面积装饰图，不做英雄宣传页。

首屏布局建议：

```text
┌────────────────────────────────────────────┐
│ 当前目标 | 最高风险 | 今日任务数 | 周期进度 │
├───────────────────────┬────────────────────┤
│ 今日任务队列          │ 指标差距/规则口径   │
│                       │ 方法论卡片          │
├───────────────────────┴────────────────────┤
│ 执行记录 / 结果 / SOP 沉淀                  │
└────────────────────────────────────────────┘
```

## 9. 第一版页面验收

详细页面规格见：

- `docs/plans/2026-05-23-1688-page-spec.md`

- 员工不用读说明就能知道今天做什么。
- 每个任务都能看到对应指标。
- 每个官方指标都能看到目标值、当前值、统计周期和来源提示。
- P0 风险不会被 P2/P3 指标淹没。
- 周复盘能把动作和指标变化连起来。
- SOP 不是独立文档，而是从任务结果中沉淀出来。
